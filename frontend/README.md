# Video Auto-Labeling — Frontend

React + TypeScript + Vite web application for viewing video segmentation results.
Deployed with AWS Amplify Gen 2.

## Features

- Video browser with search
- Interactive segmentation timeline with color-coded segments
- Click-to-seek video playback
- Segment list with type labels and timestamps
- Authenticated access (Cognito)

## Setup

```bash
npm install
npm run dev
```

### `amplify_outputs.json`

This file is auto-generated by Amplify after your first deployment. It contains your backend resource identifiers (auth pool, storage bucket, etc.) and is required for the app to connect to AWS services.

**First time setup:**
1. Deploy to Amplify (see below)
2. After deployment completes, run locally:
   ```bash
   npx ampx generate outputs --app-id <YOUR_APP_ID> --branch main
   ```
3. This creates `amplify_outputs.json` in the project root
4. The file is gitignored — each developer generates their own from the deployed backend

**For local development before first deploy:**
You can create a minimal placeholder to unblock `npm run dev`:
```json
{
  "version": "1.4",
  "auth": {},
  "storage": {}
}
```
Note: Storage and auth features won't work until you deploy and generate the real outputs.

## Deploy to Amplify

1. Push this folder to a GitHub repo
2. In AWS Amplify Console → Create new app → Connect to GitHub
3. Select the repo and branch
4. Amplify auto-detects `amplify.yml` build settings
5. Deploy
6. Generate outputs locally: `npx ampx generate outputs --app-id <APP_ID> --branch main`

## Customization

### Segment Types and Colors

Edit `src/utils/segment_types.ts` to define your segment types:

```typescript
export const SEGMENT_TYPE_COLORS = {
  'C': '#45B7D1',     // Content
  'B': '#FF6B6B',     // Break
  'T': '#4ECDC4',     // Transition
  // Add your own...
};
```

### Storage Paths

Edit `amplify/storage/resource.ts` to match your pipeline's output structure.

### Add More Pages

Add new pages in `src/pages/` and register routes in `src/App.tsx`.
